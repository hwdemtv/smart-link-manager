# 短链接工作原理与 2ms 极速跳转技术

## 技术架构总览
Smart Link Manager 采用极简的中间件架构，通过 **Edge-Native** 逻辑优化，确保全球访问延迟降至最低。

## 2ms 跳转的秘密
- **零阻塞内存缓存**：核心短码映射存储在高性能 LRU 缓存中，跳过慢速数据库查询。
- **Geo-Aware 路由**：根据用户 IP 自动选择最近的节点进行解析。
- **并发优化**：Node.js 非阻塞 I/O 处理高并发请求，确保秒杀级场景下的稳定性。

## 重定向类型详解
在 Smart Link Manager 中，您可以灵活选择重定向方式：
- **301 (永久重定向)**：最利于 SEO，将短链权重转移给目标网址。
- **302 (临时重定向)**：最常用，适用于营销活动及 A/B 测试。
- **307/308**：保持请求方法不变，适用于 API 服务。

## 数据采集过程
在跳转的毫秒瞬间，系统会自动完成以下工作：
1. 检测浏览器与操作系统类型。
2. 通过 MaxMind 数据库解析地理位置（至城市级）。
3. 识别并过滤非法爬虫与 Bot 流量。
4. 异步写入点击日志，确保不增加主跳转延迟。

## 结论
Smart Link Manager 致力于在毫秒间完成最复杂的跳转决策与数据采集。
