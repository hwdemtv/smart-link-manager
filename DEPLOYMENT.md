# Smart Link Manager 宝塔面板部署全指南 (腾讯云环境)

本项目推荐在腾讯云服务器上使用 **宝塔面板 (BT Panel)** 进行管理。为了确保“本地测试”与“生产运行”的一致性，**强烈建议优先使用 Docker 部署方式**。

---

## 🚀 方案一：Docker 部署 (推荐 - 开发/生产一致性之王)

Docker 将代码及其运行环境封装在镜像中，能彻底解决 Windows 开发环境到 Linux 生产环境常见的路径大小写、系统库库版本不兼容等问题。

### 1. 环境准备
1. 在宝塔面板 **“软件商店”** 搜索并安装 **“Docker”** 插件。
2. 确保面板已安装 **MySQL 8.0**。

### 2. 代码配置
1. 在 `/www/wwwroot/` 目录下创建项目文件夹（如 `smart-link-manager`）。
2. 将本地源码上传至该目录（**注意：排除 `node_modules` 和 `dist` 文件夹**）。

### 3. 环境变量设置
在项目根目录创建或编辑 `.env` 文件：
```env
DATABASE_URL=mysql://用户名:密码@127.0.0.1:3306/数据库名
# Docker 使用 host 网络模式，可直接连接宿主机 3306 端口
JWT_SECRET=（填写一个32位以上的随机字符串）
VITE_APP_ID=http://你的域名或服务器IP
NODE_ENV=production
PORT=3000
```

### 4. 容器构建与启动
在项目根目录下通过终端执行：
```bash
# 构建镜像并启动容器
docker-compose up -d --build

# 检查容器状态
docker ps

# 查看实时日志（用于排错）
docker logs -f smart-link-app
```

## ⚖️ 方案二：宝塔 Node 项目管理器 (轻量化方案)

如果您希望直接在宿主机运行 Node 进程，请按此步骤操作。

### 1. 基础环境
1. 宝塔 **“软件商店”** 安装 **“Node版本管理器”**。
2. 在管理器内安装 **Node.js 20.x 或 22.x**。
3. 全局安装 pnpm：`npm install -g pnpm`。

### 2. 构建项目
在项目目录下执行：
```bash
pnpm install
pnpm run build
```

### 3. 在宝塔添加项目
1. **Node项目** -> **添加项目**。
2. **项目目录**：项目源码目录。
3. **启动文件**：`dist/index.js`。
4. **环境变量**：填入 `.env` 中的所有键值对。

## 🌐 数据库架构同步 (首航关键步)

如果您的生产数据库是空的，必须同步架构。

**方案 A (推荐)**：在本地开发机执行，指向远程生产地址：
```bash
# 修改本地 .env 的 DATABASE_URL 为云数据库后执行
npx drizzle-kit push
```

**方案 B (紧急)**：在服务器终端手动补齐 Phase 12 提到的 `isDeleted` 字段。

---

## 🔒 Nginx 反反向代理与 SSL

1. 站点设置 -> **“反向代理”** -> **“添加反向代理”**。
2. **目标 URL**：`http://127.0.0.1:3000`。
3. **发送域名**：`$host`。
4. **SSL**：开启 SSL 并申请 Let's Encrypt 证书，建议强制跳转 HTTPS。

## 🛠️ 生产环境 Q&A

| 遇到问题 | 解决办法 |
| :--- | :--- |
| **500 报错（数据库相关）** | 请务必确认 MySQL 版本为 8.0。MySQL 5.7 不支持 Drizzle 的部分检查约束查询。 |
| **登录 Token 无效** | 检查服务端时间是否同步（`date` 指令），或 JWT_SECRET 是否在每次构建时被重置。 |
| **静态资源加载失败** | 确认 Nginx 反代配置中没有将 `/assets/` 拦截为错误的本地物理路径。 |
